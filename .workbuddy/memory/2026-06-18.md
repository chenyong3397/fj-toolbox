## 福建省建设行业平台 - 资质异常数据抓取（已完成）

**最终正确方案**：
1. `/credit/tendering/corps?industryId=2&outside=false` → 获取158家质量检测企业（字段名: corpName, socialCreditCode）
2. 对每家企业调用 `/credit/tendering/qualify/{corpName}` → 获取资质列表
3. 检查 `qualifyList[].abnormal` 字段：`true` = 资质异常
4. 对异常项调用 `/credit/tendering-extract/qualify?socialCreditCode=...&qualifyAssociationId=...` → 获取核查详情

**关键纠错**：
- 之前错误方案1：`forewarning/qualify-check` API 返回的是"资质核查预警"而非"资质异常"，上杭县城发等无异常企业也在其中
- 之前错误方案2：`tendering-extract` 交叉匹配socialCreditCode错误，因为企业可能在其他行业有异常资质
- 正确判断方式：qualifyInfo API返回的qualifyList中abnormal=true

**最终结果**：158家企业中10家有资质异常，共27项异常
- 泉州市恒晟工程检测有限公司（2项）✅用户确认
- 上杭县城发工程检测有限公司 不在异常列表 ✅用户确认

**技术要点**：
- 验证码：alpha通道模板匹配（100%准确率）
- pageSize范围：10-30
- 需要captcha-token header
- corps API字段名是corpName不是companyName
- tendering-extract有连字符
- 前端JS源码在app.5b072f7a.js，API定义在模块中

## 微信报告分组优化（已完成）

**需求**：同一企业内，如果多个异常资质的核查批次、核查单位、核查截止都相同，则合并显示

**修改文件**：`notify.py` 的 `generate_report_text()` 函数

**实现逻辑**：
- 按 (核查批次, 核查单位, 核查截止) 三元组分组
- 同组的资质名称用顿号拼接，标注项数（如"资质（6 项）：xxx、xxx"）
- 不同组的信息各自单独列出

**效果示例**：
- 福建省中建检测（6项异常）→ 原来6条重复的核查批次/单位/截止，现在合并为1组
- 福建正鉴检测（7项异常）→ 同理合并为1组

## 定时任务管理工具（已完成）

**需求**：用户希望有更方便的界面管理 Windows 定时任务

**创建文件**：
- `schedule_manager.py` - 交互式菜单管理工具（7个功能）
- `schedule_manager.bat` - 启动器（chcp 65001 解决中文显示）

**功能菜单**：
1. 查看任务状态（显示下次运行、上次结果、最近5行日志）
2. 安装定时任务（可自定义时间，默认08:00）
3. 修改运行时间
4. 立即运行一次（触发 schtasks /Run）
5. 删除定时任务
6. 查看运行日志（列出最近10个日志，可选择查看）
7. 测试微信通知

**技术要点**：
- 通过 `schtasks` 命令管理 Windows 任务计划程序
- 任务名：QualifyAbnormalDailyReport
- 自动查找 Python 路径（managed → fallback → sys.executable）
