## 修复 WAServiceMainContext timeout 错误

- **根因1**: `project.config.json` + `project.private.config.json` 中 `useIsolateContext: true`，开启后每个页面运行在独立 JS 上下文，配合 `getApp()` + `require()` 会导致 WAServiceMainContext 初始化超时
- **根因2**: `api.js` 中 `wx.request` 手动设置 `timeout: 8000/5000`，可能与 SDK 内部超时机制冲突
- **根因3**: `index.js` 的 `onShow()` 立即发起 API 请求，与 App 初始化竞争资源

**修复**:
- `project.config.json` + `project.private.config.json`: `useIsolateContext: true` → `false`
- `api.js`: 移除所有手动 `timeout` 参数（`request()` 和 `health()` 两处）
- `index.js`: `onShow()` 中 API 调用用 `setTimeout(..., 500)` 包裹，延迟到初始化完成后执行

  1. 命令行添加 `-u` 参数：`[py_exe, '-u', script, '--config', cfg]` — 强制无缓冲输出
  2. 新增 `cwd=project_root`：`subprocess.Popen(..., cwd=project_root)` — 确保子进程从项目根目录执行
- 验证：触发 /run API → 日志文件实时增长（1062→1845+ 字节）→ scraper 正常完成 → _最新.json scrape_time 更新为 2026-06-24 15:40:28
